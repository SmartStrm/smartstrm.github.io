# STRM 设置

在 “**系统设置**->**STRM 设置**” 中，你可以定义 `.strm` 生成的全局行为。

## 扫描设置

- **媒体文件后缀**：设置哪些文件被识别为媒体，这些文件将生成 STRM
- **媒体文件大小阈值**：仅对大于该 MB 数的文件生成 STRM，用以过滤掉过小的文件（如样片、短视频）
- **复制到本地的文件后缀**：这些文件将直接复制（下载）到生成目录，通常为 `jpg,nfo,srt` 等刮削文件

## 生成设置

### 生成根目录
- 容器中固定为 `/strm` ，一般不用更改
- 飞牛应用将跟随应用设置中的设定，注意为 SmartStrm 提供读写权限

### 基础地址
这是写入 `.strm` 文件中的前缀 URL，用以访问 SmartStrm 的直链解析服务。
- 它必须能被你的 Emby/Jellyfin 服务器访问到。
- 通常填写为 `http://宿主机IP或域名:SmartStrm管理端口`。

### URL 编码
对 `.strm` 内的 URL 进行编码。这可以增强对中文、特殊符号路径的兼容性。

### 云盘类存储生成类型
- **文件编号模式 (推荐)**：内部使用网盘的文件 ID (File ID) 作为索引。优点是请求播放时可以减少至一个请求，略微提升播放速度。
- **路径兼容模式**：使用完整的文件路径，在可能有多个请求。

### 安全签名
如果你将 SmartStrm 服务器暴露在公网，开启此功能可以防止恶意用户猜解你的文件路径。
- 外网访问时只有携带正确签名的请求才会被处理
- 启用或更换密钥后，云盘类 `.strm` 文件需要重新生成
- 当基础地址是内网IP并使用 **302代理** 时，无论签名开启或密钥更换，内部免签放行
