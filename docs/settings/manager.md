# 影视识别

在进行批量重命名或自动整理时，SmartStrm 利用 TMDB (The Movie Database) 的海量元数据来规范你的文件名。

## TMDB 配置

在 **“系统设置**->**影视识别”** 中：
- **API 密钥**：需要在前往 [TMDB - 我的 API 设置](https://www.themoviedb.org/settings/api) 获取
- **语言**：默认为 `zh-CN` 以获取中文标题和简介
- **网络代理**：由于国内网络环境原因，未设置时可能无法访问 TMDB API

## 命名模板

SmartStrm 使用极具灵活性的模板引擎，格式为：`{前缀,变量名:补位长度,后缀}`。

### 常用变量

| 变量               | 描述           | 示例               |
| :----------------- | :------------- | :----------------- |
| `{title}`          | 中文标题       | `冰与火之歌`       |
| `{title_original}` | 原始/英文标题  | `Game of Thrones`  |
| `{year}`           | 出版年份       | `2019`             |
| `{season}`         | 季号           | `01`               |
| `{episode}`        | 集号           | `01`               |
| `{quality}`        | 分辨率         | `4K` 或 `1080p`    |
| `{episode_name}`   | 剧集的小节标题 | `Winter is Coming` |
| `{tmdbid}`         | TMDB 唯一标识  | `123456`           |

### 进阶语法

1. **数字补位**：使用冒号指定长度。
   - `{episode:3}` -> `001`, `002`
2. **条件显示**：
   - `{.,episode_name}`：若有标题则显示 `.剧集标题`，否则不显示点

## 设置建议

- **剧集模板**：推荐包含 `S{season}E{episode}` 结构，这是媒体库识别季集的关键
- **综艺节目**：在 TMDB 中通常归类为 TV，同样建议使用剧集模板命名
