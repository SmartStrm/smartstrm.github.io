# 任务管理

任务是 SmartStrm 的执行核心，它实现网盘特定目录到本地 `.strm` 文件目录的映射。

我们建议为每个大类（如电影、剧集、动漫）创建独立的任务，这样可以分别执行，同时也能更好地细分管理。

## 配置任务

### 基础配置
- **任务名称**：唯一标识，建议使用英文简写（如 `movie`）
  - 生成的 `.strm` 文件将存放在 `/strm/任务名` 目录下
- **使用存储**：选择在 `存储管理` 中添加的可用存储
- **扫描路径**：存储中媒体文件所在的路径，光标在输入框停留片刻可提供目录索引

### 执行时间
- 设定自动执行的频率（分 时 日 月 周）。
- **随机按钮**：点击骰子图标可以生成一个半夜运行的随机时间点，**建议多个任务错开时间以减轻同时并非的负载**

### 执行选项

- **目录时间检查**：通过对比文件夹修改时间来跳过未变动的目录，极大提升扫描效率
  - 开启后仅当远程目录修改时间>本地目录时才扫描子目录；如文件更新后仍跳过扫描，请尝试取消本选项

- **增量生成模式**
  - 使用增量生成时，仅新增文件，不清理远端存储不存在的文件
  - 使用同步生成时，文件跟随远端，清理远端已不存在的文件
    - 同步模式下提供 **保留本地刮削的文件** 选项，可避免清理 Emby 入库时置入的刮削文件，基于常见命名规则判断，并非完全准确

- **实时监听文件变更**：在使用 "本地文件系统" 驱动时，可以开启该选项，基于文件系统变更事件实时生成对应的 `.strm` 文件，无需等待任务执行

## 工具箱

在任务编辑页，你可以点击 **“工具”** 按钮调出进阶操作：

### STRM 内容替换
当你更换了公网域名、内网 IP 或端口时，无需重新生成数万个文件，直接使用此工具一键批量替换已生成的 `.strm` 内部 URL 内容。

### 全量覆写
强制忽略本地已存在的文件，重新从网盘拉取信息生成。

### 一键清除
删除该任务目录下所有的生成文件，通常用于重置任务，或修改配置后重新生成。

## 任务插件

你可以为特定任务单独开启插件，比如让 `剧集` 任务开启 **“非法文件名修正”**，而让 `电影` 任务开启 **“飞牛刷新插件”**。这让你的任务能够高度定制化。
